Mobile Responsive Approach

How should the dashboard behave on mobile devices?

A

CSS-Only Responsive (Recommended)

Add @media queries to existing theme.py, top_bar.py, subtab_bar.py. Same codebase, same pages — just rearranged for small screens.

  • Top bar → hamburger menu on mobile
  • 3-column layouts → single column stack
  • KPI cards → 2-per-row grid
  • Tables → horizontal scroll with card view
  • Sidebar → slide-in drawer
  • Touch-friendly buttons (min 44px)
Fastest to build. No new files. Works instantly on all devices.
B

Separate Mobile Layout Mode

Detect screen width via JavaScript. Switch between desktop layout and a mobile-optimized layout with different component arrangement.

  • Bottom tab bar (like apps) instead of top nav
  • Swipe navigation between modules
  • Collapsed cards with expand on tap
  • Dedicated mobile KPI cards
  • Pull-to-refresh for data sync
More app-like feel. More work. Some features need mobile-specific code.
C

Progressive Web App (PWA)

Option A + add PWA manifest so it can be "installed" on phone home screen. Looks like a native app with splash screen and no browser chrome.

  • Everything from Option A
  • Add to Home Screen support
  • Full-screen mode (no address bar)
  • Custom app icon & splash screen
  • Offline fallback page
Best experience. Feels like a real app. Option A + small extra work.
